HubSpot Integration: Step-by-Step

HubSpot offers the most straightforward Qwin AI integration with native connector support and comprehensive API documentation.

1. Generate HubSpot API Key

  1. Log in to HubSpot: Navigate to Settings → Integrations → API Key
  2. Generate Key: Click "Generate API Key" and copy the private key
  3. Set Permissions: Ensure key has read/write access to contacts, deals, and companies
  4. Secure Key: Store key in environment variables or secret manager (never in code)

2. Configure Qwin AI → HubSpot Sync

  1. Access Integrations: In Qwin AI dashboard, go to Settings → Integrations → HubSpot
  2. Connect Account: Paste HubSpot API key and click "Connect"
  3. Map Fields: Configure field mapping between Qwin AI and HubSpot properties:
    # Field mapping example
    Qwin AI Field → HubSpot Property
    lead_email → email
    lead_name → firstname + lastname
    company_name → company
    lead_score → qwin_lead_score (custom property)
    source_url → original_source
    behavior_tags → lifecyclestage
  4. Set Sync Rules: Define when to create/update HubSpot records:
    • Create contact when lead score ≥ 30
    • Update deal stage when behavior = "demo_requested"
    • Add to list when tag = "enterprise"

3. Test & Activate

  1. Test Connection: Use "Test Sync" button to verify API connectivity
  2. Run Sample Sync: Sync 5-10 test leads to verify field mapping
  3. Monitor Logs: Check sync logs for errors or failed records
  4. Activate: Toggle integration to "Active" for production sync

HubSpot Sync Best Practices Pro Tips

✅ Use custom properties: Create HubSpot custom properties for Qwin-specific data
✅ Batch syncs: Sync in batches of 50-100 records to avoid rate limits
✅ Error handling: Configure retry logic for failed syncs
✅ Monitor usage: Track API calls to stay within HubSpot limits
✅ Document mappings: Keep field mapping documentation updated

Salesforce Integration: Advanced Setup

Salesforce requires OAuth 2.0 authentication and more complex field mapping, but offers powerful automation capabilities. For detailed guidance, see how to use Qwin AI for business process automation.

1. Create Salesforce Connected App

  1. Setup → Apps → App Manager: Click "New Connected App"
  2. Basic Info: Name: "Qwin AI Integration", API Name: "Qwin_AI_Integration"
  3. OAuth Settings: Enable OAuth, add callback URL from Qwin AI
  4. API Permissions: Select "Full access" or specific object permissions
  5. Save & Copy Credentials: Save Consumer Key and Consumer Secret

2. Configure OAuth Flow in Qwin AI

  1. Access Salesforce Integration: Settings → Integrations → Salesforce
  2. Enter Credentials: Paste Consumer Key, Consumer Secret, and Instance URL
  3. Authorize: Click "Authorize" and complete Salesforce OAuth flow
  4. Test Connection: Verify connection with sample API call

3. Advanced Field Mapping & Sync Rules

Pipedrive Integration: Quick Setup

Pipedrive offers simple webhook-based integration ideal for small teams and startups.

1. Generate Pipedrive API Token

  1. Settings → API: Navigate to API settings in Pipedrive
  2. Generate Token: Create new API token with read/write permissions
  3. Copy Token: Securely store token for Qwin AI configuration

2. Configure Webhook Sync

  1. Qwin AI Webhooks: Settings → Webhooks → Add New
  2. Endpoint URL: Enter Pipedrive webhook URL from API docs
  3. Events: Select events to sync (lead.created, deal.updated, etc.)
  4. Authentication: Add API token to webhook headers

Automation Workflows: Lead to Deal

Once integrated, build powerful automation workflows that move leads through your sales funnel automatically.

Workflow 1: Lead Capture → CRM Creation

  1. Trigger: New lead captured via Qwin AI form or chatbot
  2. AI Qualification: Qwin AI scores lead based on behavior and firmographics
  3. CRM Action: If score ≥ 40, create contact in CRM with all lead data
  4. Notification: Alert sales team via Slack/email for high-score leads
  5. Follow-up: Add lead to personalized email nurture sequence

Workflow 2: Behavioral Trigger → Deal Stage Update

  1. Trigger: Lead visits pricing page 3+ times or downloads case study
  2. AI Analysis: Qwin AI detects buying intent and updates lead score
  3. CRM Action: Move associated deal to "Qualified" or "Proposal" stage
  4. Task Creation: Auto-create follow-up task for assigned sales rep
  5. Personalization: Trigger personalized email based on behavior

Workflow 3: Deal Closed → Post-Sale Automation

  1. Trigger: Deal stage updated to "Closed Won" in CRM
  2. CRM Action: Update contact lifecycle stage to "Customer"
  3. Onboarding: Trigger welcome email sequence + onboarding tasks
  4. Analytics: Log deal data to Qwin AI for performance analysis
  5. Upsell: Add customer to upsell/cross-sell nurture sequence

Troubleshooting Common Issues

API Rate Limits

Both Qwin AI and CRMs enforce API rate limits. Mitigation strategies:

  • Batch Requests: Sync records in batches of 50-100 instead of individual calls
  • Exponential Backoff: Implement retry logic with increasing delays
  • Monitor Usage: Track API calls via dashboard alerts
  • Off-Peak Syncs: Schedule heavy syncs during low-traffic hours

Field Mapping Errors

Common causes and solutions for data sync failures:

  • Missing Required Fields: Ensure all CRM required fields are mapped
  • Data Type Mismatches: Verify field types match (text vs. number vs. date)
  • Custom Property Creation: Create custom properties in CRM before mapping
  • Validation Rules: Check CRM validation rules that might reject synced data

Authentication Issues

OAuth and API key troubleshooting:

  • Expired Tokens: Implement token refresh logic for OAuth flows
  • Permission Errors: Verify API key has required object/field permissions
  • IP Restrictions: Whitelist Qwin AI IP addresses in CRM security settings
  • Environment Variables: Store credentials securely, never in code
